# Учётные записи: админ, водитель, грузчик

## Админ уже в базе

После импорта **`schema.sql`** в базе сразу есть администратор:

| Логин | Пароль | Роль   |
|-------|--------|--------|
| **admin@localhost** | **password** | admin |

Зайди на сайт → «Войти» (или `/login`) → введи эти данные. После входа в меню появятся «Заказы» и «Админка».

---

## Водитель и грузчик

Их можно завести двумя способами.

### 1. Через сидер (тестовые логины)

Выполни в папке проекта:

```bash
php artisan db:seed
```

или только сидер с ролями:

```bash
php artisan db:seed --class=RoleUsersSeeder
```

В базе появятся (если таких email ещё нет):

| Логин | Пароль  | Роль    |
|-------|---------|---------|
| admin@localhost  | admin123 | admin   |
| driver@localhost | admin123 | driver  |
| loader@localhost  | admin123 | loader  |

Если админ был создан только из `schema.sql`, у него пароль **password**; сидер не перезаписывает существующих пользователей.

### 2. Через регистрацию и смену роли

1. Зайди на сайт как **админ** (admin@localhost).
2. Обычная регистрация: `/register` — новый пользователь создаётся с ролью **client**.
3. В phpMyAdmin открой таблицу **`users`**, найди этого пользователя и в колонке **`role`** поставь **driver** или **loader**.
4. Этот пользователь сможет входить своим email/паролем и видеть «Кабинет водителя» или «Кабинет грузчика».

---

## Итог

- **Админ**: уже в БД после импорта `schema.sql` — **admin@localhost** / **password**.
- **Водитель и грузчик**: либо создаются сидером (driver@localhost, loader@localhost с паролем admin123), либо через регистрацию + смена роли в таблице **users** в БД.
