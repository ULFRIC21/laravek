# Как работают заказы: админ, водитель, грузчик

## 1. Что лежит в базе данных

В таблице **`orders`** у каждого заказа хранится:

| Поле | Зачем |
|------|--------|
| **user_id** | Кто создал заказ (клиент или админ). Может быть пусто. |
| **from_address** | Откуда везём (адрес погрузки). |
| **to_address** | Куда везём (адрес выгрузки). |
| **weight**, **volume** | Вес и объём груза (по желанию). |
| **status** | Статус: `new` (новый), `in_progress` (в работе), `completed` (завершён). |
| **driver_id** | Кому назначен заказ как водителю — id пользователя из таблицы `users`. |
| **loader_id** | Кому назначен как грузчику — id пользователя из `users`. |
| **assigned_at** | Когда заказ назначили (дата/время). |
| **completed_at** | Когда заказ завершили. |
| **comment** | Комментарий при завершении. |

В таблице **`users`** у каждого пользователя есть поле **`role`**: `client`, `driver`, `loader`, `admin`. По нему решается, что человек видит в меню и на страницах.

Связи: один заказ связан с одним пользователем (кто создал), одним водителем (`driver_id`) и одним грузчиком (`loader_id`). В коде это задано в модели **Order**: методы `user()`, `driver()`, `loader()` — это связи к таблице `users`.

## 2. Кто что видит и как это сделано

### Админ видит «есть такой-то заказ»

- Админ заходит на **«Заказы»** (`/orders`) или на **«Админка»** (`/admin`).

**Как это устроено в коде:**

- Маршрут `/orders` вызывает **OrderController::index()**.
- В контроллере выполняется запрос к БД:
  ```php
  $orders = Order::whereIn('status', ['new', 'in_progress'])
      ->orderByDesc('created_at')
      ->get();
  ```
  То есть выбираются все заказы со статусом «новый» или «в работе», отсортированные по дате.
- В шаблон **dashboard/orders.blade.php** передаётся переменная `$orders`.
- В шаблоне по ней строится таблица: номер заказа, откуда → куда, дата, статус, кнопки «В работу» и «Завершить».

На **админке** то же самое по сути: **AdminController::index()** считает количество заказов по статусам и выбирает последние 20 заказов из БД (`Order::with(['user', 'driver', 'loader'])->limit(20)->get()`), и всё это передаётся в шаблон **dashboard/admin.blade.php**, где выводятся счётчики и таблица. То есть «такой-то заказ» админ видит потому, что контроллер читает заказы из таблицы **orders** и отдаёт их во view.

---

### Как водитель «принял» заказ (назначение водителя)

«Водитель принял» в этой схеме значит: **заказу в базе записали водителя** (поле `driver_id`).

**Шаг 1 — кто-то назначает водителя:**

- Админ (или любой с доступом к заказам) открывает заказ по кнопке **«В работу»** → попадает на страницу **исполнения заказа** (`/orders/{id}/execute`).
- Эту страницу отдаёт **OrderController::execute($id)**:
  - по `id` из БД загружается заказ: `Order::findOrFail($id)`;
  - отдельно загружаются списки пользователей с ролью `driver` и `loader`:  
    `User::where('role', 'driver')->get()` и то же для `loader`;
  - в шаблон передаются: заказ, список водителей, список грузчиков.
- На странице есть форма: выбор водителя (select), выбор грузчика (select), кнопка **«Назначить и перевести в работу»**.

**Шаг 2 — сохранение в БД:**

- Форма отправляется методом POST на маршрут **`/orders/{id}/assign`**.
- Обрабатывает **OrderController::assign(Request $request, $id)**:
  - снова загружается заказ по `id`;
  - в заказ записываются выбранные `driver_id` и `loader_id` из запроса;
  - статус заказа ставится **in_progress** («в работе»);
  - при первом назначении заполняется `assigned_at`;
  - вызывается `$order->save()` — одна строка в таблице **orders** обновляется.

После этого в базе у этого заказа есть `driver_id` = id выбранного водителя. То есть «водитель принял» — это не отдельная кнопка у водителя, а то, что админ (или менеджер) **назначил** его на заказ через эту форму; факт «принятия» — запись в БД.

---

### Почему водитель видит «свои» заказы

- Водитель заходит на **«Кабинет водителя»** (`/driver`).
- Маршрут вызывает **DriverController::index()**.
- В контроллере запрос к БД:
  ```php
  $orders = Order::where('driver_id', auth()->id())
      ->whereIn('status', ['new', 'in_progress'])
      ->orderByDesc('created_at')
      ->get();
  ```
  То есть выбираются только те заказы, у которых в поле **driver_id** записан id **текущего пользователя** (того, кто вошёл), и статус «новый» или «в работе».
- Этот список передаётся в шаблон **dashboard/driver.blade.php**, где по нему выводятся карточки «Мои рейсы».

Итого: водитель видит заказ именно потому, что в таблице **orders** у этого заказа заполнено поле **driver_id** его учётной записью. Сделано это через страницу исполнения заказа и метод **assign**.

---

### Грузчик

- Страница **«Кабинет грузчика»** (`/loader`) работает так же, только выборка по полю **loader_id**:
  ```php
  $tasks = Order::where('loader_id', auth()->id())
      ->whereIn('status', ['new', 'in_progress'])
      ->get();
  ```
- Грузчик видит заказы, на которые его **назначили** на той же странице исполнения (поле «Грузчик»).

---

### Завершение заказа

- На странице заказов есть кнопка **«Завершить»** → переход на `/orders/{id}/complete`.
- Показывается форма (страница отдаётся **OrderController::completeForm($id)**): заказ подгружается из БД по `id`, передаётся в шаблон.
- Пользователь вводит комментарий (по желанию) и нажимает «Заказ выполнен».
- Форма отправляется POST на **OrderController::complete(Request $request, $id)**:
  - заказ снова загружается по `id`;
  - в него записываются `comment` и ставится `status = completed`, `completed_at = now()`;
  - `$order->save()` обновляет строку в **orders**.

После этого заказ больше не попадает в выборки «новый/в работе» и не показывается водителю и грузчику в активных задачах (при желании для них можно отдельно выводить историю по `completed`).

## 3. Краткая цепочка «от создания до завершения»

1. **Создание заказа**  
   Кто-то (клиент или админ) на странице «Заказы» заполняет форму «Новый заказ» и нажимает «Создать».  
   → **OrderController::store()** создаёт в таблице **orders** новую строку с `status = new`, без `driver_id` и `loader_id`.

2. **Админ видит заказы**  
   **OrderController::index()** и **AdminController::index()** читают из **orders** и отдают списки в шаблоны — поэтому админ и видит «есть такой-то заказ».

3. **Назначение водителя (и грузчика)**  
   Админ нажимает «В работу» у заказа → открывается страница исполнения с формой. Выбирает водителя (и при необходимости грузчика) и нажимает «Назначить и перевести в работу».  
   → **OrderController::assign()** обновляет в БД у этого заказа `driver_id`, `loader_id`, `status = in_progress`, `assigned_at`.  
   Это и есть момент, когда «водитель принял» заказ (его id записан в заказе).

4. **Водитель видит свои заказы**  
   **DriverController::index()** выбирает из **orders** только строки, где `driver_id = auth()->id()`. Эти заказы и показываются в кабинете водителя.

5. **Завершение**  
   Кто-то с доступом нажимает «Завершить» у заказа, заполняет форму и отправляет.  
   → **OrderController::complete()** ставит заказу `status = completed`, заполняет `completed_at` и `comment`. Заказ перестаёт быть «активным» в списках и у водителя/грузчика.

Вся логика держится на одной таблице **orders** и полях **status**, **driver_id**, **loader_id**: контроллеры только читают и обновляют эту таблицу, а шаблоны показывают то, что им передали из контроллеров.
