# План: заказы, водитель, грузчик, админка

## 1. Что создать и где лежит

| Что | Где | Назначение |
|-----|-----|------------|
| **Список активных заказов** | Маршрут `/orders`, вид `resources/views/dashboard/orders.blade.php` | Таблица заказов со статусом «новый» / «в работе» с кнопками «В работу», «Завершить». |
| **Исполнение заказа** | Маршрут `/orders/{id}/execute`, вид `resources/views/dashboard/order-execute.blade.php` | Страница одного заказа: назначить водителя/грузчика, отметить «взял в работу», контакты. |
| **Завершение заказа** | Маршрут `/orders/{id}/complete`, вид `resources/views/dashboard/order-complete.blade.php` | Форма завершения: подпись/фото, комментарий, кнопка «Заказ выполнен». |
| **Страница водителя** | Маршрут `/driver`, вид `resources/views/dashboard/driver.blade.php` | Личный кабинет водителя: мои заказы, статусы, кнопки «В путь» / «Доставлено». |
| **Страница грузчика** | Маршрут `/loader`, вид `resources/views/dashboard/loader.blade.php` | Личный кабинет грузчика: заявки на погрузку/разгрузку, отметка «Выполнено». |
| **Админка** | Маршрут `/admin`, вид `resources/views/dashboard/admin.blade.php` | Сводка: заказы, пользователи, смена статусов, отчёты. |

Контроллеры можно сделать так:
- `app/Http/Controllers/OrderController.php` — список заказов, исполнение, завершение.
- `app/Http/Controllers/DriverController.php` — кабинет водителя.
- `app/Http/Controllers/LoaderController.php` — кабинет грузчика.
- `app/Http/Controllers/AdminController.php` — админ-панель.

Либо один `DashboardController` с методами по ролям и редиректом после входа на нужный URL по `user->role`.

## 2. Что должно быть на страницах (содержимое)

### Список активных заказов (`/orders`)
- Заголовок: «Активные заказы».
- Таблица: № заказа, маршрут (откуда → куда), дата, статус, действия.
- Статусы: «Новый», «В работе», «Завершён».
- Действия: «В работу» → переход на исполнение, «Завершить» → переход на завершение.
- Для «рабочего» сценария нужна таблица `orders` в БД (миграция).

### Исполнение заказа (`/orders/{id}/execute`)
- Данные заказа: маршрут, вес/объём, контакт заказчика.
- Выбор водителя (select из пользователей с ролью `driver`).
- Выбор грузчика при необходимости (select с ролью `loader`).
- Кнопки: «Назначить», «Водитель выехал», «На объекте».
- Сохранение статуса и назначений в БД.

### Завершение заказа (`/orders/{id}/complete`)
- Краткое отображение заказа.
- Поля: комментарий, подпись клиента (или чекбокс «Подтверждаю получение»).
- Кнопка «Заказ выполнен» → статус «Завершён», дата завершения.

### Страница водителя (`/driver`)
- Блок «Мои заказы»: список назначенных на текущего пользователя заказов.
- По каждому: маршрут, адрес погрузки/выгрузки, время, кнопки «В пути», «Прибыл», «Доставлено».
- История завершённых рейсов (опционально).

### Страница грузчика (`/loader`)
- Блок «Заявки на погрузку/разгрузку»: список заявок, где назначен этот грузчик.
- Адрес, время, тип (погрузка/разгрузка), кнопка «Выполнено».

### Админка (`/admin`)
- Сводка: количество заказов по статусам, список последних заказов.
- Ссылки на управление заказами, пользователями (при необходимости).
- Доступ только для пользователей с ролью `admin`.

## 3. Как сделать это рабочим

1. **Роли пользователей**  
   В таблице `users` добавить поле `role` (string: `client`, `driver`, `loader`, `admin`). Миграция:
   ```php
   Schema::table('users', function (Blueprint $table) {
       $table->string('role')->default('client')->after('email');
   });
   ```

2. **Таблица заказов**  
   Миграция `orders`: id, user_id (заказчик), from_address, to_address, weight, volume, status (new, in_progress, completed), driver_id, loader_id, assigned_at, completed_at, comment, timestamps.

3. **Middleware по ролям**  
   Создать `app/Http/Middleware/CheckRole.php`: проверка `auth()->user()->role`. В маршрутах: `Route::get('/admin', ...)->middleware(['auth', 'role:admin'])`.

4. **Роуты**  
   Группа с `middleware('auth')`:  
   - `Route::get('/orders', [OrderController::class, 'index'])->name('orders.index');`  
   - `Route::get('/orders/{id}/execute', ...)->name('orders.execute');`  
   - `Route::get('/orders/{id}/complete', ...)->name('orders.complete');`  
   - `Route::get('/driver', [DriverController::class, 'index'])->name('driver');`  
   - `Route::get('/loader', [LoaderController::class, 'index'])->name('loader');`  
   - `Route::get('/admin', [AdminController::class, 'index'])->name('admin');`  

5. **Редирект после входа**  
   В `App\Http\Controllers\Auth\LoginController` в методе `redirectTo()` или в `authenticated()` проверять `auth()->user()->role` и возвращать: админ → `/admin`, водитель → `/driver`, грузчик → `/loader`, иначе → `/orders` или `/home`.

6. **Ссылки в шапке**  
   Для авторизованных: «Личный кабинет» ведёт на `/orders` или на роль-специфичную страницу (driver/loader/admin).

## 4. Как это примерно выглядит (кратко)

- **Список заказов** — одна страница с таблицей (Bootstrap `table`), в строках кнопки «В работу» / «Завершить».
- **Исполнение заказа** — форма с полями заказа и выпадающими списками «Водитель» / «Грузчик», кнопки смены статуса.
- **Завершение заказа** — карточка заказа, текстовая область «Комментарий», кнопка «Заказ выполнен».
- **Водитель** — карточки или таблица «Мои рейсы» с кнопками «В пути», «Доставлено».
- **Грузчик** — список заявок с адресом и кнопкой «Выполнено».
- **Админка** — блоки со счётчиками и таблица последних заказов с ссылками на исполнение/завершение.

Сейчас в проекте добавлены заглушки страниц и маршруты; для полностью рабочей логики нужно подключить миграции, модели и сохранение в БД по шагам выше.
